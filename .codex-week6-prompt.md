# Week 6 — Requests Feature Implementation

You are implementing the Requests feature for a SwiftUI iOS app (iOS 17+) that connects to Seerr/Jellyseerr/Overseerr servers.

## CRITICAL RULES — READ FIRST
- Use `@Observable` macro (NOT ObservableObject) for ViewModels
- ViewModels are `@MainActor @Observable public final class`
- Repositories are `public final class` + `Sendable` (NOT @MainActor — they do I/O)
- ALL API paths come from `apiClient.endpoints` — NEVER hardcode paths
- Use `apiClient.get()`, `apiClient.post()`, `apiClient.put()`, `apiClient.deleteVoid()` etc
- `apiClient.endpoints` is `nonisolated let` — no `await` needed to read it
- Zero third-party libraries — URLSession, SwiftUI only
- Use Swift concurrency (async/await, Task) — no Combine, no completion handlers
- Files go in `SeerrClient/Features/Requests/` subdirectories

```swift
import Foundation

public final class SearchRepository: Sendable {
    private let apiClient: SeerrAPIClient

    public init(apiClient: SeerrAPIClient) {
        self.apiClient = apiClient
    }

    public func search(query: String, type: SearchType = .all, page: Int = 1) async throws -> DiscoverResponse<SearchResultItem> {
        let endpoints = apiClient.endpoints
        let path = endpoint(for: type, endpoints: endpoints)
        var queryItems = [URLQueryItem(name: "query", value: query), URLQueryItem(name: "page", value: "\(page)")]
        return try await apiClient.get(path, queryItems: queryItems)
    }
}
```

## Existing Pattern — ViewModel (follow this exactly)
```swift
import SwiftUI

@Observable
@MainActor
public final class MovieDetailViewModel {
    // MARK: - State
    enum LoadState {
        case idle
        case loading
        case loaded(MovieDetails)
        case error(String)
    }

    var loadState: LoadState = .idle
    var showRequestSheet = false

    // MARK: - Dependencies
    private let repository: MediaDetailRepository
    private let movieId: Int
    private var loadTask: Task<Void, Never>?

    // MARK: - Init
    init(movieId: Int, apiClient: SeerrAPIClient) {
        self.movieId = movieId
        self.repository = MediaDetailRepository(apiClient: apiClient)
    }

    deinit { loadTask?.cancel() }

    // MARK: - Actions
    func loadDetails() {
        guard case .idle = loadState else { return }
        loadState = .loading
        loadTask = Task {
            do {
                let details = try await repository.fetchMovieDetails(movieId: movieId)
                loadState = .loaded(details)
            } catch {
                loadState = .error(mapError(error))
            }
        }
    }
}
```

## Existing Pattern — View (follow this exactly)

```swift
import SwiftUI

struct MovieDetailView: View {
    let movieId: Int
    let movieTitle: String

    @Environment(AppState.self) private var appState
    @State private var viewModel: MovieDetailViewModel?

    var body: some View {
        Group {
            if let vm = viewModel {
                // content based on vm.loadState
            } else {
                ProgressView()
            }
        }
        .navigationTitle(movieTitle)
        .task {
            guard viewModel == nil else { return }
            guard let client = appState.apiClient else { return }
            let vm = MovieDetailViewModel(movieId: movieId, apiClient: client)
            viewModel = vm
            vm.loadDetails()
        }
    }
}
```

## IMPORTANT REMINDERS
- Import only `SwiftUI` and `Foundation` as needed
- Every file starts with: `// FileName.swift\n// SeerrClient\n//\n// Brief description.`
- Use `private` for internal helpers, `public` for init/methods that views/VMs call
- Repositories: NO @MainActor, use `Sendable`
- ViewModels: @Observable @MainActor, use Task for async work, cancel tasks in deinit
- Views: @State private var viewModel: Type? — init in .task { guard viewModel == nil }
- Error mapping: handle SeerrAPIError cases (notFound, httpError(statusCode:message:), unauthorized, etc.)
- RequestFilter is a NEW enum in RequestRepository.swift (not in SeerrModels)
- PaginatedResponse uses skip/take (not page numbers) — the API returns pageInfo with page/pages/results counts
- The GET /request endpoint accepts query params: take (Int), skip (Int), filter (String), sort (String: "added" or "modified")
