# Week 7 — Profile + Settings Implementation

You are implementing the Profile + Settings feature for a SwiftUI iOS app (iOS 17+) that connects to Seerr/Jellyseerr/Overseerr servers.

## CRITICAL RULES — READ FIRST
- Use `@Observable` macro (NOT ObservableObject) for ViewModels
- ViewModels are `@MainActor @Observable public final class`
- Repositories are `public final class` + `Sendable` (NOT @MainActor — they do I/O)
- ALL API paths come from `apiClient.endpoints` — NEVER hardcode paths
- `apiClient.endpoints` is `nonisolated let` — no `await` needed to read it
- Zero third-party libraries — URLSession, SwiftUI only
- Use Swift concurrency (async/await, Task) — no Combine, no completion handlers
- Files go in `SeerrClient/Features/Settings/` subdirectories
- Do NOT use `deinit` to cancel tasks — use a `cancelAll()` method instead (deinit is nonisolated and can't access @MainActor properties safely)
- User email must NEVER appear in logs — use IDs only
- Store Task handles for all async operations — cancel on dismiss/disappear

## Existing Pattern — ViewModel
```swift
@Observable @MainActor
public final class MovieDetailViewModel {
    enum LoadState { case idle, loading, loaded(MovieDetails), error(String) }
    var loadState: LoadState = .idle
    @ObservationIgnored private let repository: MediaDetailRepository
    @ObservationIgnored private var loadTask: Task<Void, Never>?

    public func cancelAll() { loadTask?.cancel() }  // NOT deinit

    func loadDetails() {
        guard case .idle = loadState else { return }
        loadState = .loading
        loadTask = Task {
            do {
                let details = try await repository.fetchMovieDetails(movieId: movieId)
                guard !Task.isCancelled else { return }
                loadState = .loaded(details)
            } catch {
                guard !Task.isCancelled else { return }
                loadState = .error(mapError(error))
            }
        }
    }
}
```

public enum AppTheme: Int, CaseIterable, Identifiable {
    case system = 0
    case light = 1
    case dark = 2

    public var id: Int { rawValue }

    public var displayName: String {
        switch self {
        case .system: return "System"
        case .light: return "Light"
        case .dark: return "Dark"
        }
    }

    public var colorScheme: ColorScheme? {
        switch self {
        case .system: return nil
        case .light: return .light
        case .dark: return .dark
        }
    }
}

## IMPORTANT REMINDERS
- Import only `SwiftUI` and `Foundation` as needed
- Every file starts with: `// FileName.swift\n// SeerrClient\n//\n// Brief description.`
- Use `private` for internal helpers, `public` for init/methods that views/VMs call
- Repository: NO @MainActor, use `Sendable`, all `let` properties
- ViewModel: @Observable @MainActor, use cancelAll() instead of deinit
- Views: @State private var viewModel: Type? — init in .task { guard viewModel == nil }
- Error mapping: handle SeerrAPIError cases (notFound, httpError(statusCode:message:), unauthorized, etc.)
- User privacy: NEVER log email, NEVER display email in fallback names — only in the email label
- Confirmation dialogs: `.confirmationDialog()` for sign out and disconnect
- Theme: persist as Int in UserDefaults, read in ViewModel init, apply via `.preferredColorScheme()`
- Admin check: `(user.permissions ?? 0) & 2 != 0`
- User type: 1=Plex, 2=Local, 3=Jellyfin
- BackendType has a `displayName` property (e.g. "Jellyseerr", "Overseerr", "Seerr")
- Use the existing ShimmerView from Core/Views/ShimmerView.swift for loading states
- AuthRepository is in `Features/Auth/Repositories/AuthRepository.swift` — import and use it for fetchCurrentUser/logout
- AuthRepository init: `AuthRepository(apiClient: SeerrAPIClient, server: ServerConfiguration)` — needs both apiClient and server config
- In ProfileViewModel, create AuthRepository using: `AuthRepository(apiClient: apiClient, server: appState.activeServer!)`
  - BUT guard that activeServer is non-nil before creating the VM
